Flask Project Structure – Summary

This Flask project follows the Application Factory Pattern and uses Blueprints to keep the code organized, modular, and scalable. Instead of putting everything in one file, each part of the application has a clear responsibility.

Project Structure

  • main.py – Entry point of the application. It creates and runs the Flask app.
  • website/ – The main application package.
    • __init__.py – Creates the Flask app and registers Blueprints.
    • views.py – Contains routes for general pages (e.g., home page).
    • auth.py – Contains routes related to authentication (login, signup, logout).
    • models.py – Defines database models (used later).
    • templates/ – Holds HTML files rendered by Flask.
    • static/ – Holds CSS, JavaScript, and images.

Application Factory Pattern

The Flask app is created inside a function called create_app(). This allows the application to be configured and assembled in one place, making it easier to scale, test, and manage as the project grows.

The SECRET_KEY is set inside this function to enable sessions, cookies, and authentication features.

Blueprints

Blueprints allow routes to be grouped by functionality. Each Blueprint defines routes but does not create the Flask app itself.

  • views Blueprint – Handles general pages such as the home page.
  • auth Blueprint – Handles authentication-related pages.

Blueprints are registered inside create_app(), which connects all routes to the main app.

Request Flow

  1. The application starts in main.py.
  2. create_app() creates the Flask app.
  3. Blueprints are registered.
  4. A user visits a URL.
  5. Flask finds the matching route inside a Blueprint.
  6. The route function runs and returns a response.

Key Idea

This structure separates concerns, keeps files small and readable, and prepares the application for future features such as databases, authentication, and larger codebases.
